## 프로젝트 개요
'황준호'에 관한 정보를 퀴즈로 맞추는 터미널 기반 퀴즈 게임입니다.
퀴즈 풀기, 추가, 목록 확인, 점수 확인 기능을 제공하며
프로그램을 종료해도 데이터가 유지됩니다.

## 퀴즈 주제 선정 이유
나에 대한 정보를 퀴즈로 만들어 동료들이 알아갈 수 있게 주제 선정하게 되었다.

## 실행 방법
```bash
python main.py
```

## 기능 목록
| 기능 | 설명 |
|------|------|
| 퀴즈 풀기 | 저장된 퀴즈를 순서대로 풀고 결과 확인 |
| 퀴즈 추가 | 새로운 퀴즈를 등록하고 파일에 저장 |
| 퀴즈 목록 | 등록된 퀴즈 전체 목록 확인 |
| 점수 확인 | 최고 점수 확인 |

## 데이터 파일 설명 (state.json)
- **경로**: 프로젝트 루트/state.json
- **역할**: 퀴즈 데이터와 최고 점수를 저장하여 프로그램 종료 후에도 데이터 유지
- **인코딩**: UTF-8
- **스키마**:
```json
{
    "quizzes": [
        {
            "question": "준호의 키는?",
            "choices": ["175cm", "177cm", "179cm", "181cm"],
            "answer": 3
        }
    ],
    "best_score": 80
}
```
## 코드 구조 설계 이유

### 클래스 책임 분리
- `Quiz` 클래스: 개별 퀴즈 데이터(문제, 선택지, 정답)를 표현하고 정답 확인, 출력 등 퀴즈 단위의 책임만 담당합니다.
- `QuizGame` 클래스: 게임 전체 흐름(메뉴, 퀴즈 풀기, 추가, 삭제, 저장 등)을 관리합니다.
- 두 클래스로 분리한 이유는 데이터와 로직을 분리하여 코드의 가독성과 유지보수성을 높이기 위해서입니다.

### try/except 사용 이유
- 파일 입출력 시 파일이 없거나 손상된 경우 프로그램이 비정상 종료되는 것을 방지합니다.
- 사용자 입력 시 예상치 못한 값(문자열, 빈 입력 등)으로 인한 오류를 처리합니다.
- Ctrl+C(KeyboardInterrupt), EOF 등 비정상적인 종료 상황에서도 데이터를 안전하게 저장합니다.

## JSON 저장 방식의 한계

현재 `state.json` 방식은 데이터가 적을 때는 간단하고 편리하지만, 데이터가 1000개 이상으로 늘어날 경우 아래와 같은 한계가 있습니다.

- **성능 저하**: 저장/불러오기 시 파일 전체를 읽고 써야 하므로 데이터가 많을수록 속도가 느려집니다.
- **동시 접근 불가**: 여러 사용자가 동시에 접근할 경우 데이터 충돌이 발생할 수 있습니다.
- **검색 비효율**: 특정 데이터를 찾으려면 전체를 순회해야 합니다.
- **대안**: 데이터가 많아질 경우 SQLite 등의 데이터베이스 사용을 고려할 수 있습니다.

## 파일 손상 시 복구 전략

`state.json`이 손상된 경우 아래와 같이 처리합니다.

- `json.JSONDecodeError` 또는 `KeyError` 발생 시 안내 메시지를 출력합니다.
- 손상된 파일 대신 기본 퀴즈 데이터(`DEFAULT_QUIZZES`)로 초기화합니다.
- 프로그램이 정상적으로 계속 실행될 수 있도록 합니다.

```python
except (json.JSONDecodeError, KeyError):
    print("⚠️ 데이터 파일이 손상되었습니다. 기본 퀴즈 데이터로 초기화합니다.")
    self.quizzes = DEFAULT_QUIZZES[:]
    self.best_score = 0
```
